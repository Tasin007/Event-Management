###### 2. Write the `correct answer` from the following options and give an explanation (2-5 lines).

```javascript
function sum(a, b) {
  return a + b;
}

sum(1, "2");
```

- A: `NaN`
- B: `TypeError`
- C: `"12"`
- D: `3`

<details><summary><b>Answer</b></summary>
<p>

#### Answer: A: NaN

<i>Adding a number (1) to a string ("2") using the + operator in JavaScript results in concatenation, which produces the string "12". This is then interpreted as a numeric addition, resulting in NaN.</i>

</p>
</details>

###### 3. Write the `correct answer` from the following options and give an explanation (2-5 lines).

```javascript
const food = ["ğŸ•", "ğŸ«", "ğŸ¥‘", "ğŸ”"];
const info = { favoriteFood: food[0] };

info.favoriteFood = "ğŸ";

console.log(food);
```

- B: `['ğŸ', 'ğŸ«', 'ğŸ¥‘', 'ğŸ”']`
- C: `['ğŸ', 'ğŸ•', 'ğŸ«', 'ğŸ¥‘', 'ğŸ”']`

<details><summary><b>Answer</b></summary>
<p>

#### Answer: A: `['ğŸ•', 'ğŸ«', 'ğŸ¥‘', 'ğŸ”']`

<i>The food array is not modified in the code. The info.favoriteFood property is initially set to the first element of the food array, which is "ğŸ•". Later, info.favoriteFood is changed to "ğŸ", but this does not affect the original food array. Therefore, when you log food, it remains ['ğŸ•', 'ğŸ«', 'ğŸ¥‘', 'ğŸ”'].</i>

</p>
</details>

###### 4. Write the `correct answer` from the following options and give an explanation (2-5 lines).

```javascript
function sayHi(name) {
  return `Hi there, ${name}`;
}

console.log(sayHi());
```

- A: `Hi there,`
- B: `Hi there, undefined`
- C: `Hi there, null`
- D: `ReferenceError`

<details><summary><b>Answer</b></summary>
<p>

#### Answer: B: `Hi there, undefined`

<i>The sayHi function expects an argument name, but it's called without one. JavaScript interprets the missing argument as undefined, so it returns "Hi there, undefined".</i>

</p>
</details>

});

- B: 2
- C: 3
- D: 4

<p>

#### Answer: B: 2

<i>The forEach loop counts elements in the nums array that are truthy (not equal to 0). Only 1, 2, and 3 are truthy, so count becomes 2.</i>
